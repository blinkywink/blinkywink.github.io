Character Device

Character Device

Creates a character that can be customized and set in a pose.
PlayerTrigger

Memory

(Varies) (Base 810)

Data Layers

Not Supported

Overview

The Character Device is a Device in Fortnite: Creative, added in Update v18.20.

See Gallery for a list of some of the available characters.

All default values have been Bolded.

All Options
Option Value Description
Character Creative Mannequin, Select a Character Select a Character to be used.
Use Animated Idle On, Off Determines if the idle pose is animated.
Idle (Pose) Jazz Hands 1, Anguish, Leap, Thumbs Up, Standing Tall, Finger Guns, Run - Determined, Kick the Ball, Salute, Flex, Jazz Hands 2, Love Hands, Kneeling Crouch, Survey, Hands on Hips 1, Lift Off, Ready to Fight, Run, Strut, Floating, Wave, Hand Stop, Hand Stop, Floating - Look Back, Peace Sign, Hands on Hips 2 Sets the Character Device to a static idle pose.
Emote Sit In Chair, Air Guitar, Ground Pound, Beckon, Air Horn, Cheer, Clap, Nod Head, Phone Wave, Point, Shake Head, Wave Set the Character Device to this emote (non-looping) when triggered.
Interact Type Do Not Interact, Send Event Only Can the player interact with this character device? If so, it will transmit a on the defined channel.
Interact Text Interact, {Custom Text} Can the player interact with this character device? If so, it will transmit a on the defined channel.
Interact Time Instant, 0.1 - 20 Seconds Determines the length of interaction required to activate the device.
Visible During Game On, Off Determines whether the device is visible during the game. This does affect its collision properties.
Character Slot 0, 1 - 19 Allows Character Device Controllers to affect Character Devices with matching Character Slot values.
Generate Overlap Events Yes, No Determines whether the Character Device will generate overlap collision events with other devices.

Functions and Events

Functions and Events[]

Functions
Option Description
Turn On Visibility When Receiving From Shows the character device, enables collision and allows interaction (if enabled).
Turn Off Visibility When Receiving From Hides the character device and disables collision. Players will not be able to interact with the device while hidden.
Enable When Receiving From Enables the device on receiving a signal from the selected channel.
Play Audio When Receiving From Plays the emote when receiving from channel.
Disable When Receiving From Disables the device on receiving a signal from the selected channel.
Events
Option Description
On Interacted With Send Event To When a player interacts with the character device, send a signal on the channel.
